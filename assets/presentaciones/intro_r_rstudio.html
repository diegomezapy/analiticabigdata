¿Qué es R?

Definición

R es un lenguaje de programación estadístico y entorno de análisis de datos de código abierto, desarrollado por Ross Ihaka y Robert Gentleman en 1993 en la Universidad de Auckland, Nueva Zelanda.

Características principales

  • Especializado en estadística: regresión, series temporales, ML, visualización
  • Vectorizado por naturaleza: opera sobre vectores y matrices directamente
  • +20 000 paquetes en CRAN para casi cualquier tarea analítica
  • Gráficos de calidad publicación con ggplot2, plotly, leaflet
  • Reproducible: scripts, R Markdown, Quarto para informes automáticos
  • Multiplataforma: Windows, macOS, Linux

¿Para qué se usa R en este curso?

  • Carga y limpieza de datos (tidyverse, data.table)
  • Análisis exploratorio (EDA) y visualización
  • Estadística descriptiva e inferencial
  • Machine learning supervisado y no supervisado
  • Evaluación de modelos (caret, tidymodels)
  • Informes reproducibles con R Markdown
  • Aplicaciones web con Shiny

Dato histórico

R es una implementación del lenguaje S (Bell Labs, 1976). La letra "R" refleja tanto los apellidos de sus creadores como esa herencia. Hoy es mantenido por el R Core Team y la fundación R Foundation.

¿Qué es RStudio?

Definición

RStudio es un IDE (Entorno de Desarrollo Integrado) para R. No es R en sí mismo, sino una interfaz gráfica que hace que trabajar con R sea mucho más cómodo y productivo.

Desarrollado por Posit

La empresa Posit PBC (antes RStudio Inc.) mantiene el IDE. Existe versión Desktop gratuita y versión Server/Cloud. También desarrollan tidyverse, Quarto y Shiny.

¿R sin RStudio?

Sí, es posible usar R desde la consola base. Pero RStudio agrega autocompletado, gestión de proyectos, visualización integrada, depuración y mucho más.

Paso 1: Instalar R

1
Ir a CRAN (Comprehensive R Archive Network) Visitar el repositorio oficial de R: cran.r-project.org
2
Seleccionar tu sistema operativo Elegir Download R for Windows → base → Download R-4.x.x for Windows
3
Ejecutar el instalador Aceptar configuración predeterminada. R se instala en C:\Program Files\R\R-4.x.x\
4
Verificar instalación Abrir la consola base de R o CMD y ejecutar R --version
⬇ Descargar R para Windows   ⬇ Descargar R para macOS

Versiones disponibles

  • R 4.4.x (recomendada, estable) — compatibilidad máxima con paquetes
  • Se actualizan 2–3 veces por año (versiones menores trimestrales)
  • Para proyectos de aula: cualquier R ≥ 4.2 es suficiente

Requisitos mínimos

  • Windows 10 / 11 (64-bit recomendado)
  • macOS 11 Big Sur o superior
  • RAM: 4 GB mínimo, 8 GB recomendado
  • Disco: ~200 MB para R base; paquetes extra ~1–2 GB
  • Conexión a internet para instalar paquetes desde CRAN

Nota importante

Instalar primero R, después RStudio. RStudio detecta automáticamente la instalación de R al iniciarse. Si instalas en orden inverso, deberás configurar la ruta manualmente.

Diapositiva 9 de 14 · Fundamentos

Tipos de datos y estructuras en R

Tipos básicos — class() y typeof()
# Tipos de datos básicos
class(3.14) # "numeric"
class(5L) # "integer"
class("hola") # "character"
class(TRUE) # "logical"
class(NA) # "logical" (valor faltante)

# Estructuras de datos
vec <- c(1, 2, 3) # vector
mat <- matrix(1:9, 3, 3) # matriz
df <- data.frame(x=c(1,2), y=c("a","b")) # data frame
lst <- list(n=42, s="hola") # lista
fac <- factor(c("A","B","A")) # factor
Indexación y operaciones
# Indexar vectores (empieza en 1, no 0)
notas <- c(85, 72, 91, 68, 95)
notas[1] # primer elemento → 85
notas[2:4] # elementos 2 al 4
notas[notas > 80] # filtro lógico

# Manejo de NA
datos <- c(1, NA, 3, NA, 5)
is.na(datos) # detectar faltantes
mean(datos, na.rm=TRUE) # ignorar NA → 3
sum(is.na(datos)) # contar NA → 2

# Columnas de un data frame
df$columna # acceso con $
df[df$x > 1, ] # filtrar filas
Diapositiva 10 de 14 · Paquetes

Instalar y cargar paquetes

Gestión de paquetes
# Instalar un paquete (solo una vez)
install.packages("ggplot2")
install.packages(c("dplyr", "tidyr"))

# Cargar el paquete (en cada sesión)
library(ggplot2)
library(dplyr)

# Ver paquetes instalados
installed.packages()[, "Package"]

# Actualizar todos los paquetes
update.packages(ask = FALSE)

# Instalar desde GitHub (con remotes)
install.packages("remotes")
remotes::install_github("usuario/repo")

# Usar función sin cargar el paquete
ggplot2::ggplot(data, aes(x, y))

CRAN — El repositorio oficial

El Comprehensive R Archive Network contiene más de 21 000 paquetes revisados y probados. install.packages() descarga directamente desde CRAN.

install.packages() vs library()

FunciónCuándoFrecuencia
install.packages()Descargar e instalarUna vez
library()Activar en sesiónCada script
update.packages()Actualizar versionesPeriódica

Error común

Error: there is no package called 'ggplot2'

Significa que olvidaste instalar el paquete. Ejecutar primero install.packages("ggplot2") y luego library(ggplot2).

Diapositiva 12 de 14 · Práctica

Tu primer script completo en R

mi_primer_script.R — Análisis demográfico Paraguay
# ── 1. Cargar paquetes ─────────────────────────
library(ggplot2)
library(dplyr)

# ── 2. Crear datos ─────────────────────────────
departamentos <- data.frame(
  depto = c("Central", "Alto Paraná", "Itapúa",
           "Caaguazú", "San Pedro"),
  pob = c(2310000, 850000, 560000,
           475000, 320000)
)

# ── 3. Exploración básica ──────────────────────
str(departamentos) # estructura
summary(departamentos) # estadísticas
glimpse(departamentos) # vista rápida (dplyr)

# ── 4. Visualizar ─────────────────────────────
ggplot(departamentos, aes(x = reorder(depto, pob),
                        y = pob/1e6)) +
  geom_col(fill = "#14b8a6", width = 0.7) +
  coord_flip() +
  labs(title = "Población por departamento",
       x = "", y = "Millones de habitantes") +
  theme_minimal()

Flujo típico de un script R

  • 1. library() — cargar paquetes al inicio
  • 2. Importar datos — read_csv(), read_excel(), read.csv()
  • 3. Explorar — str(), summary(), head(), dim()
  • 4. Limpiar — filter(), mutate(), na.omit()
  • 5. Analizar — modelos, tests estadísticos
  • 6. Visualizar — ggplot2, plotly
  • 7. Exportar — write_csv(), ggsave(), rmarkdown

Buenas prácticas

  • Guardar el script con Ctrl+S frecuentemente
  • Usar comentarios # ── para secciones
  • Nombres descriptivos: datos_limpios en lugar de d2
  • Un proyecto RStudio por análisis
  • Nunca usar setwd() en scripts compartidos
