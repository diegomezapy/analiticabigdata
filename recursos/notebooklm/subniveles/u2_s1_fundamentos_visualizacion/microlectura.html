2.1 Fundamentos de Visualización de Datos

1. Por qué visualizamos

John Tukey, padre del análisis exploratorio, decía que el propósito del análisis de datos era el insight, no los números. La visualización es el puente entre los datos crudos y la comprensión humana.

El célebre cuarteto de Anscombe ilustra por qué: cuatro datasets con medias, varianzas y correlaciones idénticas pero con distribuciones radicalmente distintas. Sin visualización, el analista concluiría que son iguales.

2. Gramática de Gráficos

Leland Wilkinson formalizó en 2005 los componentes de cualquier gráfico estadístico:

  • Datos: el dataset que se mapea.
  • Estéticas (aes): variables mapeadas a propiedades visuales (x, y, color, size, shape).
  • Geometrías (geom): la representación visual (punto, línea, barra, área).
  • Estadísticas (stat): transformaciones opcionales (conteo, suavizado, media).
  • Facetas: subgráficos para subgrupos.
  • Coordenadas: cartesianas, polares, proyecciones.
  • Tema: apariencia no relacionada con datos.

ggplot2 en R implementa esta gramática directamente. Cada capa es una función que modifica el gráfico base.

3. Codificación visual efectiva

Cleveland y McGill (1984) establecieron la jerarquía de precisión perceptual:

  1. Posición en escala común
  2. Posición en escala no común
  3. Longitud
  4. Ángulo / pendiente
  5. Área
  6. Volumen / curvatura
  7. Densidad / saturación de color

La regla práctica: usar posición o longitud para variables cuantitativas; color o forma para variables categóricas.

4. EDA: el proceso

El Análisis Exploratorio no tiene un protocolo rígido, pero sigue estas etapas:

  1. Univariado: distribución de cada variable por separado (histogramas, boxplots, frecuencias).
  2. Bivariado: relaciones entre pares de variables (dispersión, correlación, tablas cruzadas).
  3. Multivariado: patrones entre 3+ variables (matrices de dispersión, heatmaps de correlación).
  4. Anomalías: identificar outliers, valores faltantes, inconsistencias.
  5. Hipótesis: formular preguntas que el análisis confirmatorio responderá.

5. Errores comunes

  • Eje Y que no empieza en cero en barras (exagera diferencias)
  • Pie charts con más de 5 categorías (ilegible)
  • 3D innecesario (distorsiona percepción)
  • Paletas de color no aptas para daltónicos
  • Demasiada información en un solo gráfico

6. Buenas prácticas

Usar paletas accesibles (viridis, ColorBrewer). Siempre incluir título, ejes etiquetados y fuente de datos. Para audiencias no técnicas, preferir gráficos simples y anotaciones explicativas sobre sofisticación visual.
