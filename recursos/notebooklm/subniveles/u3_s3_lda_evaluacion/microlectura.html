3.3 LDA y Evaluación de Modelos

1. La idea geométrica de LDA

LDA busca proyectar los datos a un espacio de menor dimensión donde las clases estén lo más separadas posible. Imagina puntos de dos clases en un espacio 2D: LDA encuentra la línea sobre la cual proyectar los puntos de manera que la distancia entre las proyecciones de cada clase sea máxima y la dispersión dentro de cada clase sea mínima.

Matemáticamente, LDA maximiza el cociente de dispersión entre clases / dispersión dentro de clases, lo que Fisher (1936) formalizó como el criterio de Fisher.

2. LDA como clasificador

El clasificador LDA asigna un punto nuevo a la clase con mayor probabilidad a posteriori usando el teorema de Bayes, bajo los supuestos de normalidad y covarianzas iguales. Esto produce fronteras de decisión lineales (planos en 3D, hiperplanos en general).

Cuando las covarianzas no son iguales entre clases, se usa QDA (Quadratic DA) que produce fronteras cuadráticas.

3. La trampa del overfitting y el data leakage

Un error común en ML es evaluar el modelo en los mismos datos usados para entrenarlo — obteniendo resultados demasiado optimistas. El data leakage ocurre cuando información del test set "se filtra" al entrenamiento.

Forma más común de leakage: normalizar todos los datos juntos antes de dividir en train/test. La media y SD del test set influencian la normalización del train set. La corrección: normalizar primero el train set, luego aplicar la misma transformación al test set.

4. Validación cruzada: la solución robusta

La validación cruzada k-fold divide el dataset en k partes iguales (folds). En cada iteración, uno de los k folds es el test y los k-1 restantes son el train. Se repite k veces y se promedian los resultados.

Ventajas: todo el dataset se usa tanto para entrenamiento como para validación; la estimación del rendimiento tiene menor varianza que una sola división. Con k=10, es el estándar de la industria.

5. La matriz de confusión y sus derivados

La matriz de confusión muestra las 4 combinaciones de predicción vs realidad:

  • TP (True Positive): predicho positivo, real positivo.
  • TN (True Negative): predicho negativo, real negativo.
  • FP (False Positive, Tipo I): predicho positivo, real negativo.
  • FN (False Negative, Tipo II): predicho negativo, real positivo.

Precision = TP/(TP+FP): de los predichos positivos, cuántos son realmente positivos. Recall = TP/(TP+FN): de los realmente positivos, cuántos fueron detectados. F1 = armónica entre Precision y Recall.

6. Comparación de múltiples modelos

Para comparar modelos de forma rigurosa: usar validación cruzada en el mismo conjunto de folds para todos los modelos; comparar con intervalos de confianza; usar tests estadísticos como el test de McNemar para diferencias significativas; considerar también el tiempo de entrenamiento y la interpretabilidad.
